# AGENTS.md

このファイルは、AIエージェント（LLM）がこのリポジトリ（fmrs）を理解し、開発や修正を支援するためのコンテキストとガイドラインを提供するものです。

## 1. プロジェクト概要
**プロジェクト名**: fmrs (Shogi Helpmate Solver)
**目的**: 将棋の「協力詰（ばか詰）」専用の高速なソルバー。
**特徴**: 
- ブラウザ上（WebAssembly経由）で完全に動作するためサーバサイドが存在しない。
- 非常に長手数の問題（例：寿限無 19447手）を数秒で解くパフォーマンスを誇る。
- コマンドラインツールとしても利用可能。
- **一本道詰将棋の自動生成**（`one_way_mate`）機能の開発が進行中。

## 2. ディレクトリ構成と技術スタック
このプロジェクトは大きく分けて、フロントエンド（React/TS）とコアロジック（Rust/Wasm）から構成されています。

- **`rust/`**: Rustのワークスペースディレクトリ。
  - **`rust/fmrs_core/`**: 協力詰ソルバーのコアロジック（局面表現、SFENのエンコード/デコード、合法手生成、探索アルゴリズムなど）。
  - **`rust/src/`**: CLIツールとしてのエントリポイントやコマンド定義（`main.rs`, `command/`など）。特に `command/one_way_mate/` には一本道詰将棋の自動生成アルゴリズム（ビームサーチ等）が実装されている。
  - **`rust/wasm/`**: ブラウザ上で実行するためのWebAssemblyバインディング (`wasm-bindgen`を利用)。
- **`app/`**: Webフロントエンドのコード（TypeScript + React）。
  - `app/src/`: Reactコンポーネント群、Wasmとのインターフェース、状態管理。
  - Webpack（`@wasm-tool/wasm-pack-plugin`）を使用してWasmビルドとフロントエンドビルドを統合している。

## 3. ドメイン知識・用語
- **協力詰（ばか詰 / Helpmate）**: 攻め方と玉方が協力して玉を詰める将棋のルール。通常の詰将棋とは探索の前提条件（王手義務や最善手の概念）が異なる点に注意。
- **一本道詰将棋 (One-way mate)**: 変化や紛れがなく、解（手順）がただ一つに定まる協力詰問題。本リポジトリではビームサーチを用いた自動生成手法の研究・実装（`one_way_mate` コマンド）が行われている。**注意: この自動生成においては「初形の美しさ」や「作品としての不自然さのなさ」といった審美的な要素は一切考慮しておらず、ひたすら「長手数であること（ステップ数の最大化）」のみを目標としています。**
- **SFEN**: 将棋の盤面を文字列で表現する記法。初期局面のパースなどに使われる。
- **Zobrist Hashing / Bitboard**: `fmrs_core` 内での高速な盤面管理や局面の同一性判定に用いられているデータ構造。

## 4. 開発・ビルドのワークフロー
エージェントがコードを修正する際は、以下のコマンドで動作確認を行ってください。

### Rust (コアロジック・CLI・Wasm)
- ワーキングディレクトリ: `rust/`
- ビルド・実行: `cargo run -r solve standard <sfen>`
- テスト: `cargo test` (ロジック修正時は必須)
- 静的解析・フォーマット: `cargo clippy`, `cargo fmt`

### Webフロントエンド
- ワーキングディレクトリ: プロジェクトルート (または `app/` に依存)
- 依存関係のインストール: `npm install`
- 開発サーバ起動: `npm run serve` (実行にはシステムに `wasm-pack` がインストールされている必要があります)
- 本番ビルド: `npm run build`
